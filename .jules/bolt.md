## 2024-05-24 - Optimizing high-frequency updates in React lists
**Learning:** In lists with high-frequency updates (like `DownloadsPage` updating progress from a backend listener), avoiding unnecessary re-renders of the entire list and untouched list items is critical for performance. Previously, list items would re-render entirely when any other item's progress changed because the parent's state change triggered re-renders down the tree.
**Action:** Apply `React.memo` to the individual list item components (`DownloadCard`, `DownloadHistoryCard`, `SearchHistoryCard`) and use `useCallback` on all handler functions passed as props to ensure they have stable references. This prevents items whose specific props haven't changed from re-rendering during frequent state updates of siblings.
