## 2024-03-01 - React.memo custom comparison for high-frequency progress updates
**Learning:** In list views where items receive high-frequency progress updates (like a downloads list), mapping a separate `progressMap` and merging it into the props of list items causes the entire list container (`DownloadsPage`) to re-render constantly. Without `React.memo` and a custom comparison function (`arePropsEqual`) on the list item component (`DownloadCard`), every item re-renders even if its specific progress or status hasn't changed, leading to severe performance degradation.
**Action:** When implementing lists with real-time progress or state updates, wrap the list item component with `React.memo` and use a precise custom comparison function to ensure it only re-renders when its specific data changes. Also, ensure all event handlers passed to the list item are wrapped in `useCallback` to prevent reference changes from triggering re-renders.
