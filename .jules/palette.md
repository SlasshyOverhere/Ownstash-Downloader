## 2024-05-24 - Accessible Toggle Switches
**Learning:** Custom interactive UI elements in this repository (e.g., custom Toggles built from divs/buttons) often lack proper ARIA roles (like `role="switch"`), dynamic `aria-checked` states, explicit `aria-label`, and `focus-visible` classes for keyboard navigation. This prevents screen readers from announcing their state and purpose correctly, and makes them difficult to interact with via keyboard alone.
**Action:** When implementing or updating custom interactive elements like Toggles, always ensure semantic HTML features are emulated. Add `role="switch"`, dynamically bind `aria-checked`, pass down `aria-label` or link to an existing label via `aria-labelledby`, and provide visible focus styles using tailwind's `focus-visible:` utilities.
