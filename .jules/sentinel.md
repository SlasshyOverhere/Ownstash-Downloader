## 2025-02-28 - Path Traversal in open_folder Tauri Command
**Vulnerability:** In `src-tauri/src/commands.rs`, the `open_folder` command took an optional `file_name` string from the frontend and directly appended it to `base_path` via `.join(name)`. This allowed path traversal (e.g., passing `../../Windows/System32/cmd.exe` as `file_name`) because `std::path::PathBuf::join` replaces the current path completely if the joined path is absolute, or climbs up directories if `..` is provided.
**Learning:** Never directly join an untrusted string to a base path in Rust without verifying or sanitizing it. Even if `base_path.join()` looks safe, it does not trap the resulting path inside `base_path` when dealing with traversal sequences or absolute overrides.
**Prevention:** Sanitize untrusted path components by using `std::path::Path::new(untrusted_str).file_name().and_then(|f| f.to_str()).unwrap_or("")`. This extracts just the final filename component, safely dropping any directory traversal characters.
