## 2024-05-18 - Prevent ACE via PATH Manipulation in Custom Player Settings
**Vulnerability:** The `open_with_external_player` command directly invoked `Command::new` using user-provided paths from the frontend settings. This allowed an attacker or malicious script running in the frontend to supply paths like `cmd.exe` or `powershell.exe` which resolved automatically via system PATH, leading to Arbitrary Code Execution (ACE) bypassing file existence checks.
**Learning:** Checking if a file exists (`Path::new(&player).exists()`) does not prevent execution of system utilities present in the PATH if the input is not an absolute path, because `Command::new` will fallback to searching the PATH environment variable for bare executable names.
**Prevention:** Always enforce that user-supplied executable paths are absolute (`is_absolute()`) before passing them to `Command::new`, or use Tauri's scoped `shell` plugin configuration to restrict external command execution to pre-approved binaries.
