## 2024-05-31 - Path Traversal in `open_folder` (Tauri Command)
**Vulnerability:** The `open_folder` command in `src-tauri/src/commands.rs` accepted an optional `file_name` parameter which was directly joined with the `base_path` via `base_path.join(name)`. This allowed an attacker or malicious script to supply a filename containing path traversal characters like `../../../` to execute operations outside of the designated `base_path` (e.g., highlighting or executing a file in `C:\Windows\System32` or `/etc/passwd`).
**Learning:** In the Rust backend, filenames extracted from external sources or frontend parameters must be sanitized before being used with `PathBuf::join()`. `PathBuf::join()` does not automatically sanitize directory traversal sequences (`..`).
**Prevention:** Always sanitize user-provided filenames using `std::path::Path::new(&name).file_name()` before combining them with other paths. To safely fallback to an empty string instead of raw input, use `.and_then(|f| f.to_str()).unwrap_or("")`.
